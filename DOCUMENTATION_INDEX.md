### Essential Commands

```bash
# Start platform
docker-compose up -d

# Check status
docker-compose ps

# View logs
docker-compose logs -f api

# Stop platform
docker-compose down

# Rebuild after code changes

```

- **API:** http://localhost:8000
- **API Docs:** http://localhost:8000/docs

- **Health Check:** http://localhost:8000/health

## üéì Documentation Best Practices

### When Writing New Docs

1. **Start with the user:** Who will read this?
2. **Be concise:** Get to the point quickly
3. **Use examples:** Show, don't just tell
4. **Add timestamps:** Document when it was written
5. **Link generously:** Connect related docs
6. **Test commands:** Ensure they work
7. **Update index:** Add to this file

### Documentation Standards

- **Headers:** Use emoji for visibility
- **Code blocks:** Specify language (```python, ```bash)
- **Lists:** Use for scannability
- **Tables:** For comparisons
